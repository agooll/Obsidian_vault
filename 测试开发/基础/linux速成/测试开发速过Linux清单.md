# 测试开发速过 Linux 清单

## 一、目标先定清楚

我学 Linux，不是为了走运维方向，也不是为了背一大堆命令。  
我现在学 Linux 的目标是：

1. 能进入项目目录并运行测试脚本
2. 能看日志，辅助定位问题
3. 能查进程，判断服务是否正常
4. 能做最基本的文件、权限处理
5. 面试时能回答“Linux 在测试开发里怎么用”

---

## 二、最必须掌握的 6 组内容

## 1. 定位位置：知道我现在在哪、目录里有什么

### 必须会的命令

```bash
pwd
ls
ls -l
cd 目录名
cd ..
cd ~
```

### 它们是干什么的

- `pwd`：查看当前所在目录
- `ls`：查看当前目录下有什么文件
- `ls -l`：查看详细信息
- `cd xxx`：进入某个目录
- `cd ..`：返回上一级目录
- `cd ~`：回到用户主目录

### 测试开发里怎么用

- 进入接口自动化项目目录
- 找日志目录
- 找配置文件目录
- 切换到报告输出目录

### 你至少要会到什么程度

- 能在 Linux 里切目录不迷路
- 给你一个项目路径，你能进去
- 给你一个日志路径，你能进去

---

## 2. 文件操作：增删改移动复制

### 必须会的命令

```bash
mkdir test_dir
touch a.txt
cp a.txt b.txt
mv b.txt c.txt
rm c.txt
rm -r test_dir
```

### 它们是干什么的

- `mkdir`：创建目录
- `touch`：创建文件
- `cp`：复制文件
- `mv`：移动 / 重命名文件
- `rm`：删除文件
- `rm -r`：删除目录

### 测试开发里怎么用

- 新建测试目录
- 复制配置文件
- 重命名日志文件
- 删除旧报告
- 清理临时文件

### 你至少要会到什么程度

- 能创建和删除简单文件
- 能复制一个配置文件到新位置
- 能把日志文件改名保存

### 注意

```bash
rm -rf
```

这个命令杀伤力很大，知道它的作用就行，别乱删。

---

## 3. 看文件和日志：这是最重要的

### 必须会的命令

```bash
cat xx.log
less xx.log
tail -f xx.log
tail -n 20 xx.log
grep error xx.log
grep -i error xx.log
grep -n error xx.log
```

### 它们是干什么的

- `cat`：直接查看整个文件内容
- `less`：分页查看文件
- `tail -f`：动态追踪日志
- `tail -n 20`：看最后 20 行
- `grep`：搜索关键字
- `-i`：忽略大小写
- `-n`：显示行号

### 测试开发里怎么用

- 接口失败后看服务日志
- 查 `error`、`exception`、`fail`
- 看程序启动报错
- 看 pytest 执行后的输出日志
- 看线上/测试环境的应用日志

### 你至少要会到什么程度

- 能看某个日志文件最后 20 行
- 能搜索日志里的 error
- 能持续追踪日志变化

### 最常用组合

```bash
tail -f app.log
grep -n error app.log
tail -n 50 app.log
```

---

## 4. 查进程：服务活没活着

### 必须会的命令

```bash
ps -ef
ps -ef | grep python
ps -ef | grep pytest
top
kill -9 进程ID
```

### 它们是干什么的

- `ps -ef`：查看系统中的进程
- `ps -ef | grep xxx`：查某个进程在不在
- `top`：动态看系统资源和进程状态
- `kill -9 PID`：强制结束进程

### 测试开发里怎么用

- 查服务有没有启动成功
- 查自动化脚本是不是还在跑
- 查某个进程是不是卡住了
- 查 CPU 占用高的程序

### 你至少要会到什么程度

- 能查一个服务进程是否存在
- 能查 pytest 是否还在执行
- 能结束一个异常卡死的进程

### 常见表达

```bash
ps -ef | grep 服务名
```

这个是面试里很常说的一句。

---

## 5. 查找文件：日志、配置、脚本在哪

### 必须会的命令

```bash
find . -name "*.log"
find . -name "pytest.ini"
find / -name "app.log"
```

### 它是干什么的

- `find`：按照名字查找文件或目录

### 测试开发里怎么用

- 找日志文件
- 找 pytest 配置文件
- 找测试报告
- 找某个 shell 脚本

### 你至少要会到什么程度

- 能找到 `.log`
- 能找到 `pytest.ini`
- 能找到某个配置文件

---

## 6. 权限：为什么脚本跑不了 / 文件打不开

### 必须会的命令

```bash
ls -l
chmod +x run.sh
chmod 644 a.txt
chmod 755 run.sh
```

### 它们是干什么的

- `ls -l`：查看文件权限
- `chmod +x`：给脚本执行权限
- `chmod 644`：常见文件权限
- `chmod 755`：常见脚本/目录权限

### 测试开发里怎么用

- shell 脚本无法执行时
- 自动化脚本权限不足时
- 日志文件没有读取权限时

### 你至少要会到什么程度

- 能看懂 `ls -l`
- 知道“权限不够”会导致运行失败
- 能给脚本加执行权限

### 你现阶段记住这一句就够了

```bash
chmod +x run.sh
```

因为这句最常用。

---

## 三、测试开发里最常见的 Linux 实战场景

### 场景 1：自动化脚本跑不起来

你会这样做：

1. `pwd` 看自己在哪
2. `ls` 看目录里有没有脚本
3. `ls -l` 看脚本有没有执行权限
4. `chmod +x run.sh`
5. 重新执行

---

### 场景 2：接口失败，怀疑是后端问题

你会这样做：

1. 找到日志文件
2. `tail -n 50 app.log`
3. `grep -n error app.log`
4. 看有没有异常报错
5. 必要时 `tail -f app.log` 动态观察

---

### 场景 3：服务没起来

你会这样做：

1. `ps -ef | grep 服务名`
2. 看进程是否存在
3. 没有的话去看启动日志
4. 有的话再结合日志判断为什么不可用

---

### 场景 4：CPU 很高，怀疑程序卡住

你会这样做：

1. `top`
2. 看哪个进程占用高
3. 再 `ps -ef | grep xxx`
4. 判断是否需要结束异常进程

---

## 四、你现在可以先不深学的内容

下面这些知道名字就行，先别花太多时间：

- shell 脚本深度编写
- 用户组复杂管理
- Linux 内核原理
- 网络配置细节
- systemctl / service 深度运维
- 文件系统原理
- 大量复杂参数组合

原因很简单：  
你现在目标是测试开发入门，不是 Linux 运维岗。

## 七、你现阶段的过关标准

不是背 100 个命令。  
而是做到下面这 5 件事：

1. 能切到指定目录
2. 能找到日志文件
3. 能看日志里的报错
4. 能查服务进程在不在
5. 能在 Linux 下跑一次 pytest

只要做到这 5 条，对测试开发初学阶段就已经够用了。

**测试开发快速学 Linux，先只抓 6 件事：目录操作、文件操作、日志查看、进程排查、文件查找、基础权限。重点不是背命令，而是能辅助你跑测试、看日志、查问题。**
