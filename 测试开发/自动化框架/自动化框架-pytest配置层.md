# 自动化框架 pytest 配置层

`pytest.ini` 的作用是统一配置 pytest 行为。这个示例里主要配置的是日志系统。

## 1. 当前配置做了什么

它同时配置了两类日志输出：

- 控制台日志
- 文件日志

对应配置项包括：

- `log_file`
- `log_file_level`
- `log_file_format`
- `log_file_date_format`
- `log_cli`
- `log_cli_level`
- `log_cli_format`
- `log_cli_date_format`

## 2. 文件日志的意义

```ini
log_file = ./log/test.log
```

表示测试执行日志会落盘到 `log/test.log`。

这样做的价值是：

- 测试结束后还能回溯问题
- 方便定位失败时间点
- 方便保存历史执行记录

## 3. 控制台日志的意义

```ini
log_cli = True
```

表示执行 pytest 时，日志同步打印到控制台。

这样调试时可以直接看到：

- 当前执行到哪条用例
- 哪一步报错
- 哪一步请求失败

## 4. 日志格式说明

当前格式里包含：

- 时间
- 文件名
- 函数名
- 行号
- 日志级别
- 日志内容

这说明作者已经在往“可调试、可排查”的方向走，而不是只看断言通过或失败。

## 5. 这一层在自动化框架里的意义

很多人只把自动化框架理解成：

- 发请求
- 做断言

但真正可维护的框架还必须有配套的配置层，至少包括：

- 日志
- 环境
- 路径
- 报告
- 编码规范

这个示例里 `pytest.ini` 先承担了日志配置职责。

## 6. 当前配置的小问题

源码里日志格式中的中文出现了编码异常，说明文件编码或查看环境有兼容问题。逻辑不受影响，但笔记里要知道这是“显示异常”，不是配置含义异常。

## 7. 一句话总结

`pytest.ini` 属于自动化框架的配置层，当前主要负责统一日志输出规则，让测试执行结果更容易观察和排查。
