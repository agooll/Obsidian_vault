## 1. 根目录文件

### `run.py`

职责：

- 统一启动 pytest
- 生成 allure 原始结果
- 生成 allure HTML 报告

定位：

- 入口层

源码：

```python
import os
import pytest

if __name__ == "__main__":
    pytest.main(["-vs", "./testcases/test_runner.py", "--alluredir", "./report/json_report", "--clean-alluredir"])
    os.system("allure generate ./report/json_report -o ./report/html_report --clean")
```

解析：

- `pytest.main(...)` 直接在 Python 里启动 pytest
- `-vs` 表示显示详细信息并允许 `print`
- `--alluredir` 把执行结果输出到 allure 原始报告目录
- `os.system(...)` 再把原始结果生成 HTML 报告

### `demo.py`

import requests

### `utils/excel_utils.py`

职责：

- 读取 Excel
- 读取第二行字段名
- 从第三行开始取测试数据
- 组装成 `list[dict]`
- 过滤 `is_true` 为假的用例

定位：

- 工具层

源码：

```python
import openpyxl

def read_excel():
    workbook = openpyxl.load_workbook("./data/测试用例.xlsx")
    worksheet = workbook["Sheet1"]

    data = []
    keys = [cell.value for cell in worksheet[2]]
    for row in worksheet.iter_rows(min_row=3, values_only=True):
        dict_data = dict(zip(keys, row))
        if dict_data["is_true"]:
            data.append(dict_data)

    workbook.close()
    return data
```

解析：

- 读取 `Sheet1`
- 第二行作为字段名
- 第三行开始读数据
- `dict(zip(keys, row))` 把一行数据转成一条字典
- 最后返回 `list[dict]`

### `testcases/test_runner.py`

职责：

- 通过 pytest 参数化执行 Excel 用例
- 组装请求
- 发起接口调用
- 做 HTTP 断言和数据库断言
- 做 JSON 提取和 SQL 提取
- 维护全局变量池完成接口关联

定位：

- 执行层

核心源码：

```python
class TestRunner:
    data = read_excel()
    all = {}

    @pytest.mark.parametrize("case", data)
    def test_case(self, case):
        all = self.all
        allure.dynamic.feature(case["feature"])
        case = eval(Template(str(case)).render(all))

        method = case["method"]
        url = "http://192.168.10.131:8888/api/private/v1" + case["path"]
        headers = eval(case["headers"]) if isinstance(case["headers"], str) else None
        params = eval(case["params"]) if isinstance(case["params"], str) else None
        data = eval(case["data"]) if isinstance(case["data"], str) else None
        json = eval(case["json"]) if isinstance(case["json"], str) else None
        files = eval(case["files"]) if isinstance(case["files"], str) else None

        request_data = {
            "method": method,
            "url": url,
            "headers": headers,
            "params": params,
            "data": data,
            "json": json,
            "files": files,
        }

        res = requests.request(**request_data)
```

断言和提取源码：

```python
        if case["check"]:
            assert jsonpath.jsonpath(res.json(), case["check"])[0] == case["expected"]
        else:
            assert case["expected"] in res.text

        if case["jsonExData"]:
            for key, value in eval(case["jsonExData"]).items():
                value = jsonpath.jsonpath(res.json(), value)[0]
                all[key] = value
```

数据库校验与提取源码：

```python
        if case["sql_check"] and case["sql_expected"]:
            conn = pymysql.Connect(
                host="192.168.10.131",
                port=3306,
                database="mydb",
                user="root",
                password="123456",
                charset="utf8"
            )
            cur = conn.cursor()
            cur.execute(case["sql_check"])
            result = cur.fetchone()
            cur.close()
            conn.close()
            assert result[0] == case["sql_expected"]
```

解析：

- `data = read_excel()` 先把所有 Excel 用例读进来
- `@pytest.mark.parametrize` 让每一行用例都独立执行一次
- `Template(...).render(all)` 用于替换 `{{TOKEN}}` 这类关联变量
- `requests.request(**request_data)` 统一发起请求
- HTTP 断言、数据库断言、JSON 提取、SQL 提取都在这一层完成
- `all = {}` 是全局变量池，前置接口提取的值会给后续接口复用

## 3. 这个框架的最小闭环

从源码清单看，它的执行逻辑可以压缩成一句话：

> Excel 定义用例，`excel_utils.py` 读取数据，`test_runner.py` 统一执行请求和断言，`run.py` 负责启动并产出报告。

## 3.1 按源码看执行顺序

如果按真正运行顺序去理解，这套框架是这样串起来的：

1. `run.py` 调用 `pytest.main(...)`
2. pytest 加载 `testcases/test_runner.py`
3. `test_runner.py` 在类加载阶段执行 `data = read_excel()`
4. `utils/excel_utils.py` 读取 Excel，返回 `list[dict]`
5. `@pytest.mark.parametrize("case", data)` 把每一行数据展开成一个测试实例
6. `test_case()` 内部先渲染 `{{TOKEN}}` 等模板变量
7. 再拼接请求参数，调用 `requests.request(...)`
8. 然后做 HTTP 断言、数据库断言
9. 最后把 `jsonExData`、`sqlExData` 提取到全局变量池 `all`
10. pytest 执行完后，`run.py` 再调用 allure 生成 HTML 报告

这个顺序一旦记住，后面再看自动化框架扩展，比如封装配置文件、封装 request 层、封装数据库层，就不会乱。
