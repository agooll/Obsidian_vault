#### 1.项目应该如何熟悉？
1. 先搞清楚这个项目是干嘛的 弄明白这个项目存在的意义
2. 弄明白这个项目是给谁用的 确定系统有哪些人用 管理员 内部员工 外部客户 等等
3. 拆解模块 看看系统里面都有哪些功能点
4. 看技术栈 了解一下它是用什么做的，比如前端是 Vue 还是 React，后端是 Java 还是 Python，数据库用的什么？
#### 2.WMS项目的简单介绍？
1. ==项目定位==:WMS项目他是一个仓库管理系统 它是一个B/S架构(浏览器访问)的系统 主要是优化仓库的管理和优化了仓库的操作流程 
2. ==核心功能==:它集成了仓库管理的各种环节，简单说就是：**采购 -> 入库 -> 存储 -> 出库 -> 承运**，还包括中间的盘点等节点
3. ==使用人群==： 这个系统是给采购、销售、客服和仓管员这些角色用的
4. ==技术实现==： 它是典型的前后端分离架构。前端用的是 Vue + ElementUI，后端是 Spring Boot 框架，数据库用了 MySQL
