#### 1.接口自动化中 四种请求的方式有什么区别？四种请求方式的写法是怎么样的？
`get` 和 `delete` 是在==请求行==中发送数据的 他们用的是`params`关键字发送==键值对==数据
`post` 和 `put` 是在==请求体==中发送数据的 他们可以用==data关键字(传键值对)==发送数据或者是==json关键字(传JSON)==发送数据
写法：
```
get
requests.get(url,params = {"key","value"})
delete
requests.delete(url,params = {"key","value"})
post
requests.post(url,data = {"key","value"})
requests.post(url,json = {"key","value"})
put
requests.put(url,data = {"key","value"})
requests.put(url,put = {"key","value"})
```
对比：

| 请求方法   | 对应关键字     | 参数位置      | 数据格式     |
| ------ | --------- | --------- | -------- |
| get    | params    | 请求行(URL)  | 键值对      |
| delete | params    | 请求行(URL)  | 键值对      |
| post   | data/json | 请求体(body) | 键值对/json |
| put    | data/json | 请求体(body) | 键值对/json |

#### 2.响应体的状态码和cookie以及响应体如何转换成json格式的数据?用代码实现
**获取响应状态码**
先用requests库定义响应对象 然后使用`对象名.status_code`的方法 获取响应体中的状态码 
**获取响应cookie**
在获取响应后 可以通过响应对象的 `.cookies` 属性来获取 
**转换为json()**
在获取到响应对象后可以直接使用`.json()` 方法转换成json格式的数据
代码：
```

url = "https://api.example.com/login"
request_data = {
    "method": "POST",
    "url": url,
    "json": {"username": "testuser", "password": "123"} # 使用 json 提交请求体 [2]
}

```
