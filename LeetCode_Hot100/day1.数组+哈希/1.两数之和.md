### 题目
```TXT
给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 **和为目标值** _`target`_  的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

**示例 1：**

**输入：**nums = [2,7,11,15], target = 9
**输出：**[0,1]
**解释：**因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。

**示例 2：**

**输入：**nums = [3,2,4], target = 6
**输出：**[1,2]

**示例 3：**

**输入：**nums = [3,3], target = 6
**输出：**[0,1]

**提示：**

- `2 <= nums.length <= 104`
- `-109 <= nums[i] <= 109`
- `-109 <= target <= 109`
- **只会存在一个有效答案**
```
### 暴力解法 时间复杂度是O(n²)
#### ACM写法
```C++
#include<iostream>
#include<vector>
using namespace std;
int main() {
    int n;
    int target;
    cin >> n >> target;
    vector<int> nums(n);
    for (int i = 0; i < n; i++)
    {
        cin >> nums[i];
    }
    for (int i = 0;i<n;i++)
    {
        for (int j = i + 1;j< n;j++)
        {
            if (nums[i] + nums[j] ==target)
            {
                cout << "[" << nums[i] << "," << nums[j] << "]" << endl;
            }
        }
   }
    return 0;
}
```
#### LeetCode写法
```C++
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        int n = nums.size();
        for(int i = 0;i<n;i++)
        {
            for(int j = i+1;j<n;j++)
            {
                if(nums[i]+nums[j]==target)
                {
                    return{i,j};
                }
            }
        }
        return {};
    }
};
```

### 哈希表解法 时间复杂度是O(n)

#### 核心思路
- 一边遍历数组，一边用哈希表记录“某个数值出现过的位置”。
- 当前遍历到 `nums[i]` 时，先看 `target - nums[i]` 之前有没有出现过。
- 如果出现过，说明已经找到答案。
- 如果没有，再把当前值和下标存进哈希表。

#### 为什么是 O(n)
- 数组只遍历一遍。
- `unordered_map` 的查找和插入平均时间复杂度都是 `O(1)`。
- 所以总时间复杂度是 `O(n)`。

#### ACM写法
```C++
#include <iostream>
#include <vector>
#include <unordered_map>
using namespace std;

int main() {
    int n, target;
    cin >> n >> target;

    vector<int> nums(n);
    for (int i = 0; i < n; i++) {
        cin >> nums[i];
    }

    unordered_map<int, int> mp;

    for (int i = 0; i < n; i++) {
        int need = target - nums[i];
        if (mp.count(need)) {
            cout << mp[need] << " " << i << endl;
            return 0;
        }
        mp[nums[i]] = i;
    }

    return 0;
}
```

#### LeetCode写法
```C++
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> mp;

        for (int i = 0; i < nums.size(); i++) {
            int need = target - nums[i];
            if (mp.count(need)) {
                return {mp[need], i};
            }
            mp[nums[i]] = i;
        }

        return {};
    }
};
```

#### `mp.count(need)` 是什么

是的，`count` 可以理解成 `unordered_map` 的一个常用成员函数（API）。

这里：

```C++
mp.count(need)
```

意思就是：

```TXT
去哈希表 mp 里检查 key = need 的这一项存不存在
```

在这题里：

- 存在这个 key，返回 `1`
- 不存在这个 key，返回 `0`

所以代码：

```C++
if (mp.count(need))
```

本质上就是：

```TXT
如果 need 这个值之前已经在哈希表里出现过
```

#### 为什么这里要用 `count`

因为这题只是想先判断：

```TXT
前面有没有出现过这个 need
```

这时候用 `count` 最直接。

#### 为什么不直接写 `mp[need]`

因为：

```C++
mp[need]
```

如果这个 key 原来不存在，`unordered_map` 会自动新建一个键值对。

这样会把哈希表“顺手改掉”，不适合单纯做存在性判断。

而：

```C++
mp.count(need)
```

只是检查，不会乱插入新数据。

#### 一句话记忆

```TXT
count(x) = 查 key = x 在哈希表里存不存在
```

#### 和暴力解法的区别
- 暴力解法：两层循环，时间复杂度 `O(n^2)`。
- 哈希表解法：一层循环 + 哈希查找，时间复杂度 `O(n)`。
- 这题面试里更推荐直接说哈希表解法。
