### 题目
```TXT
给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。

**示例 1:**

**输入:** strs = ["eat", "tea", "tan", "ate", "nat", "bat"]

**输出:** [["bat"],["nat","tan"],["ate","eat","tea"]]

**解释：**

- 在 strs 中没有字符串可以通过重新排列来形成 `"bat"`。
- 字符串 `"nat"` 和 `"tan"` 是字母异位词，因为它们可以重新排列以形成彼此。
- 字符串 `"ate"` ，`"eat"` 和 `"tea"` 是字母异位词，因为它们可以重新排列以形成彼此。

**示例 2:**

**输入:** strs = [""]

**输出:** [[""]]

**示例 3:**

**输入:** strs = ["a"]

**输出:** [["a"]]

**提示：**

- `1 <= strs.length <= 104`
- `0 <= strs[i].length <= 100`
- `strs[i]` 仅包含小写字母
```

### 题解思路（核心总结）

先创建一个哈希表，key 是字符串排序后的结果，value 是一个` vector<strng>`。  
遍历每个原字符串时，先把它排序，得到一个标准形式 key。  
如果两个字符串排序后的 key 一样，说明它们的字符组成完全相同，只是顺序不同，所以它们属于同一组。  
然后把原字符串放进这个 key 对应的 `vector<string>`'里。  
最后把哈希表里的所有 value 取出来，就是答案。

#### 这题本质在做什么

这题不是找一个值，而是做“分组”。

我们要把：

```TXT
字符组成完全相同、只是顺序不同的字符串
```

放进同一组。

#### 最核心的做法

1. 遍历每一个原字符串 `s`
2. 复制出一份 `key = s`
3. 对 `key` 排序
4. 排序后的 `key` 作为分组标识
5. 把原字符串 `s` 放进 `mp[key]` 对应的那一组里

#### 一定要注意的点

- 不是把“所有单词整体排序一次”
- 而是把“每一个单词单独排序”
- 如果两个单词排序后的结果一样，说明它们属于同一组

例如：

```TXT
eat -> aet
tea -> aet
ate -> aet
```

所以：

```TXT
"aet" -> ["eat", "tea", "ate"]
```

#### 哈希表为什么这样设计

```C++
unordered_map<string, vector<string>> mp;
```

表示：

- `key`：排序后的字符串，例如 `"aet"`
- `value`：这一组里的所有原字符串，例如 `["eat", "tea", "ate"]`

#### 你可以这样理解这题

```TXT
先给每个单词算一个“统一标签”（排序后的字符串），
标签相同的，就放进同一个 vector 里。
```

#### 一句话版

```TXT
每个单词单独排序，排序结果相同，就归到同一个 key 下面；value 里存的是这一组的原字符串。
```

### ACM解法
```C++
#include <iostream>
#include <vector>
#include <string>
#include <unordered_map>
#include <algorithm>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<string> strs(n);

    for (int i = 0; i < n; i++) {
        cin >> strs[i];
    }

    unordered_map<string, vector<string>> mp;

    for (int i = 0; i < n; i++) {
        string s = strs[i];

        string key = s;
        sort(key.begin(), key.end());

        mp[key].push_back(s);
    }

    for (auto& p : mp) {
        for (int i = 0; i < p.second.size(); i++) {
            cout << p.second[i];

            if (i + 1 < p.second.size()) {
                cout << " ";
            }
        }
        cout << endl;
    }

    return 0;
}
```

### ACM解法（逐行注释版）
```C++
#include <iostream>                 // 引入输入输出流库，后面要用 cin 和 cout
#include <vector>                   // 引入 vector 容器
#include <string>                   // 引入 string 字符串类型
#include <unordered_map>            // 引入哈希表 unordered_map
#include <algorithm>                // 引入 sort 排序函数
using namespace std;                // 使用标准命名空间，后面就不用反复写 std::

int main() {                        // 主函数，程序从这里开始执行
    int n;                          // 定义一个整数 n，表示有多少个字符串
    cin >> n;                       // 从输入中读入 n

    vector<string> strs(n);         // 创建一个长度为 n 的字符串数组 strs

    for (int i = 0; i < n; i++) {   // 循环 n 次，依次读入每个字符串
        cin >> strs[i];             // 把第 i 个输入字符串存到 strs[i] 中
    }

    unordered_map<string, vector<string>> mp; // 定义哈希表：key 是排序后的字符串，value 是同组异位词列表

    for (int i = 0; i < n; i++) {   // 再遍历一遍所有字符串，开始分组
        string s = strs[i];         // 取出当前字符串，保存到 s 中

        string key = s;             // 复制一份当前字符串到 key，后面要对 key 排序
        sort(key.begin(), key.end()); // 对 key 排序，让异位词得到相同的“特征串”

        mp[key].push_back(s);       // 把原字符串 s 放进 key 对应的那一组里
    }

    for (auto& p : mp) {            // 遍历哈希表中的每一组，p 表示一个键值对
        for (int i = 0; i < p.second.size(); i++) { // 遍历这一组中的每个字符串
            cout << p.second[i];    // 输出当前这一组中的第 i 个字符串

            if (i + 1 < p.second.size()) { // 如果后面还有字符串
                cout << " ";        // 就输出一个空格，方便分隔
            }
        }
        cout << endl;               // 当前这一组输出完成后换行
    }

    return 0;                       // 程序正常结束，返回 0
}
```

### ACM写法里几个最容易卡住的点

#### 1. 为什么要先写 `int n; cin >> n;`

因为 ACM 模式下，程序不会像 LeetCode 一样直接把 `vector<string> strs` 传给你。

你需要自己先读入“后面一共有多少个字符串”。

例如输入可能是：

```TXT
6
eat tea tan ate nat bat
```

这里第一行的 `6` 就是 `n`。

#### 2. `vector<string> strs(n);` 的意义是什么

这一句的意思是：

```C++
创建一个长度为 n 的字符串数组
```

如果 `n = 6`，那相当于先准备好 6 个位置：

```C++
strs[0], strs[1], strs[2], strs[3], strs[4], strs[5]
```

这样后面你才能写：

```C++
cin >> strs[i];
```

因为 `strs[i]` 这个位置必须先存在，才能往里面放值。

#### 3. 为什么下面要这样读输入

```C++
for (int i = 0; i < n; i++) {
    cin >> strs[i];
}
```

这段代码的作用是：

- 第 1 次循环，把第 1 个字符串读到 `strs[0]`
- 第 2 次循环，把第 2 个字符串读到 `strs[1]`
- ...
- 第 n 次循环，把第 n 个字符串读到 `strs[n-1]`

也就是说，最后 `strs` 里面就存好了全部输入字符串。

#### 4. 为什么要写 `unordered_map<string, vector<string>> mp;`

这题不是找一个数，而是要把很多字符串分组。

所以需要一个结构来表示：

```TXT
某个特征值 -> 这一组里的所有字符串
```

这里：

- `key`：排序后的字符串，例如 `"aet"`
- `value`：属于这一组的所有原字符串，例如 `["eat", "tea", "ate"]`

所以才设计成：

```C++
unordered_map<string, vector<string>> mp;
```

也就是：

```TXT
一个 string key，对应一个 vector<string> value
```

#### 5. 为什么 value 可以是 `vector<string>`

因为一个分组里不止一个字符串。

例如：

```TXT
"aet" -> ["eat", "tea", "ate"]
```

如果 value 只是一个 `string`，那就只能存一个单词，没法存整组。

所以这里必须用：

```C++
vector<string>
```

来装这一组里的多个字符串。

#### 6. 为什么 `key` 要用排序后的字符串

因为字母异位词只是顺序不同，但字符组成相同。

更准确地说：

- 不是把所有单词一起排序
- 而是每个单词各自单独排序
- 只要排序后的结果一样，就说明它们是同一组

例如：

```TXT
eat -> aet
tea -> aet
ate -> aet
```

它们排序后都一样，所以排序后的结果就可以当成“这一组的统一标识”。

##### 补充1：这里的 `key` 不是关键字

`key` 只是你自己起的一个普通变量名，不是 C++ 保留关键字。

例如下面这样写也完全可以：

```C++
string t = s;
sort(t.begin(), t.end());
```

这里只是因为 `key` 这个名字更容易看出它表示“分组标签”，所以才这样命名。

##### 补充2：`string key = s;` 是什么意思

这句表示：

```TXT
把原字符串 s 复制一份给 key
```

这样后面排序时改的是 `key`，不是原来的 `s`。

例如：

```TXT
s = "eat"
key = "eat"
```

排序后：

```TXT
s 还是 "eat"
key 变成 "aet"
```

##### 补充3：`key.begin(), key.end()` 是哪里到哪里

```C++
sort(key.begin(), key.end());
```

表示：

```TXT
把 key 这个字符串的所有字符都拿出来排序
```

如果：

```TXT
key = "eat"
```

那排序的就是字符：

```TXT
e, a, t
```

排完后得到：

```TXT
"aet"
```

#### 7. `mp[key].push_back(s);` 到底在干什么

这句可以拆成两部分理解：

- `mp[key]`：找到 `key` 对应的那一组
- `push_back(s)`：把当前字符串 `s` 放进这一组里

例如：

```C++
string s = "eat";
string key = "aet";
mp[key].push_back(s);
```

执行后就变成：

```TXT
"aet" -> ["eat"]
```

再来一个 `tea`：

```TXT
"aet" -> ["eat", "tea"]
```

所以这句的本质就是：

```TXT
把当前字符串放到它应该去的那一组里
```

#### 8. 为什么这里的 for 看起来和普通 for 不一样

你这题里有两种 `for`。

第一种是普通 for：

```C++
for (int i = 0; i < n; i++)
```

这种是“按下标遍历”。

第二种是范围 for：

```C++
for (string s : strs)
for (auto& pair : mp)
```

这种是“直接从容器里一个一个取元素”。

例如：

```C++
for (string s : strs)
```

意思就是：

```TXT
把 strs 里的每个字符串，依次取出来给 s
```

而：

```C++
for (auto& pair : mp)
```

意思就是：

```TXT
把 mp 里的每个键值对，依次取出来给 pair
```

然后：

- `pair.first` 表示 key
- `pair.second` 表示 value

##### 补充1：这里的 `pair` 不是关键字

这里的 `pair` 只是你自己起的变量名，不是 C++ 保留关键字。

要注意区分两件事：

- `pair<const string, vector<string>>`：这里的 `pair` 是一种**类型**，表示“键值对类型”
- `for (auto& pair : mp)`：这里的 `pair` 是你自己起的**变量名**，表示“当前遍历到的这个键值对”

也就是说：

- 截图里那种 `pair<...>`，问的是“这个元素的数据类型是什么”
- `for` 里面那个 `pair`，问的是“我给当前元素起了什么名字”

你完全可以写成：

```C++
for (auto& p : mp)
for (auto& item : mp)
for (auto& entry : mp)
```

效果都一样。

##### 补充2：为什么会有 `first` 和 `second`

因为 `unordered_map` 里的每个元素，本质上都是一个键值对：

```C++
pair<const string, vector<string>>
```

所以：

- `first` 表示键，也就是 `key`
- `second` 表示值，也就是 `value`

#### 这里的操作逻辑是什么

当你写：

```C++
for (auto& pair : mp)
```

意思就是：

```TXT
把哈希表 mp 里的每一个键值对，依次取出来，临时命名为 pair
```

然后：

- `pair.first` 就是这一项的 key
- `pair.second` 就是这一项的 value

例如如果当前这一项是：

```TXT
"aet" -> ["eat", "tea", "ate"]
```

那这时候：

- `pair.first = "aet"`
- `pair.second = ["eat", "tea", "ate"]`

所以代码：

```C++
ans.push_back(pair.second);
```

意思就是：

```TXT
把当前这一组 value 整体放进最终答案 ans
```

例如：

```TXT
"aet" -> ["eat", "tea", "ate"]
```

那就对应：

- `pair.first = "aet"`
- `pair.second = ["eat", "tea", "ate"]`

#### 9. 为什么这里写 `auto&`

- `auto`：让编译器自动推导类型
- `&`：表示引用，不额外拷贝一份数据

因为 `unordered_map<string, vector<string>>` 的元素类型其实很长：

```C++
pair<const string, vector<string>>
```

手写很麻烦，所以一般直接写：

```C++
for (auto& p : mp)
```

#### 10. 这题 ACM 写法的整体流程

1. 先读入字符串个数 `n`
2. 再把所有字符串读进 `strs`
3. 遍历每个字符串，排序后得到 `key`
4. 用 `mp[key].push_back(s)` 分组
5. 最后遍历哈希表，把每组输出出来

#### 一句话总结

这题 ACM 写法的本质就是：

```TXT
先自己把输入读进来，再用“排序后的字符串”当 key，用哈希表把同类字符串分到同一组
```

### LeetCode写法
```C++
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        unordered_map<string, vector<string>> mp;

        for (string s : strs) {
            string key = s;
            sort(key.begin(), key.end());

            mp[key].push_back(s);//将key相同的不同s存入mp
        }

        vector<vector<string>> ans;

        for (auto& pair : mp) {
            ans.push_back(pair.second);
        }

        return ans;
    }
};
```
