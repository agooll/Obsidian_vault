### 题目
```txt
给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

**示例 1：**

**输入：**nums = [100,4,200,1,3,2]
**输出：**4
**解释：**最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

**示例 2：**

**输入：**nums = [0,3,7,2,5,8,4,6,0,1]
**输出：**9

**示例 3：**

**输入：**nums = [1,0,1,2]
**输出：**3

**提示：**

- `0 <= nums.length <= 105`
- `-109 <= nums[i] <= 109`
```
### 原思路

### 题解思路（从0到1理解）

#### 1. 题目到底要你做什么

这题要找的是：

```TXT
最长的“数值连续”序列长度
```

注意：

- 不要求这些数在原数组里相邻
- 只要求数值上连续

例如：

```TXT
[100,4,200,1,3,2]
```

虽然 `1,2,3,4` 在原数组里不是挨着的，
但它们数值连续，所以最长长度是 `4`。

#### 2. 为什么排序法不是最优

最容易想到的方法是：

1. 先排序
2. 再扫一遍，统计最长连续段

这个方法能做出来，但复杂度是：

```TXT
O(n log n)
```

因为排序本身就要 `O(n log n)`。

而题目要求：

```TXT
O(n)
```

所以要换思路。

#### 3. 这题的核心想法

我们真正需要的是：

```TXT
快速判断某个数字在不在
```

比如当前数是 `4`，我们只关心：

- `3` 在不在
- `5` 在不在

如果这个判断很快，这题就能做。

#### 4. 为什么容器选 `unordered_set`

##### 不选 `vector`

因为如果你想知道：

```TXT
某个数字在不在 nums 里
```

用 `vector` 查找，一般要从头扫到尾，复杂度是：

```TXT
O(n)
```

太慢。

##### 不优先选 `unordered_map`

因为这题不需要：

```TXT
key -> value
```

的映射关系。

我们只关心：

```TXT
这个数存不存在
```

所以 `unordered_map` 也能做，但有点多余。

##### 最适合的是 `unordered_set`

因为它刚好适合：

- 存一堆数字
- 快速判断某个数字在不在
- 自动去重

时间复杂度平均是：

- 插入 `O(1)`
- 查找 `O(1)`

所以这题最合适的容器是：

```C++
unordered_set<int>
```

#### 5. 为什么“只从起点开始找”

这是这题最关键的一步。

比如数组里有：

```TXT
1,2,3,4
```

如果你从每个数都往后找：

- 从 `1` 找 `2,3,4`
- 从 `2` 再找 `3,4`
- 从 `3` 再找 `4`

就会重复计算很多次。

所以正确做法是：

```TXT
只从连续序列的起点开始往后找
```

##### 什么叫起点

如果当前数是 `x`：

- 如果 `x - 1` 不存在，说明 `x` 是起点
- 如果 `x - 1` 存在，说明 `x` 不是起点，跳过

例如：

```TXT
1,2,3,4
```

- `1` 前面没有 `0`，所以 `1` 是起点
- `2` 前面有 `1`，所以 `2` 不是起点
- `3` 前面有 `2`，所以 `3` 不是起点

这样就不会重复统计。

#### 6. 手动模拟一遍

输入：

```TXT
[100,4,200,1,3,2]
```

先放进集合：

```TXT
{100,4,200,1,3,2}
```

开始遍历：

- `100`
  - `99` 不在集合里
  - 所以 `100` 是起点
  - 往后看 `101` 不在
  - 长度是 `1`

- `4`
  - `3` 在集合里
  - 所以 `4` 不是起点，跳过

- `200`
  - `199` 不在集合里
  - 所以 `200` 是起点
  - 往后看 `201` 不在
  - 长度是 `1`

- `1`
  - `0` 不在集合里
  - 所以 `1` 是起点
  - 往后找：`2` 在，`3` 在，`4` 在，`5` 不在
  - 长度是 `4`

所以答案就是：

```TXT
4
```

#### 7. LeetCode写法

```C++
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        unordered_set<int> st(nums.begin(), nums.end());
        int ans = 0;

        for (int x : st) {
            if (!st.count(x - 1)) {
                int cur = x;
                int len = 1;

                while (st.count(cur + 1)) {
                    cur++;
                    len++;
                }

                ans = max(ans, len);
            }
        }

        return ans;
    }
};
```

#### 8. LeetCode写法逐步解释

##### `unordered_set<int> st(nums.begin(), nums.end());`

把数组中的所有数字放进哈希集合里。

作用：

- 方便后面快速判断某个数字在不在
- 自动去重

##### `int ans = 0;`

记录最终答案，也就是最长连续序列的长度。

##### `for (int x : st)`

遍历集合中的每个数字。

##### `if (!st.count(x - 1))`

判断 `x` 是不是某段连续序列的起点。

意思是：

- 如果 `x-1` 不存在
- 那 `x` 才是起点

##### `int cur = x; int len = 1;`

- `cur`：当前往后扩展到哪个数字
- `len`：当前这一段连续序列长度

##### `while (st.count(cur + 1))`

只要下一个数还存在，就继续往后扩展。

##### `ans = max(ans, len);`

用当前这一段长度更新最大值。

#### 9. ACM写法

```C++
#include <iostream>
#include <vector>
#include <unordered_set>
#include <algorithm>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<int> nums(n);
    for (int i = 0; i < n; i++) {
        cin >> nums[i];
    }

    unordered_set<int> st(nums.begin(), nums.end());
    int ans = 0;

    for (int x : st) {
        if (!st.count(x - 1)) {
            int cur = x;
            int len = 1;

            while (st.count(cur + 1)) {
                cur++;
                len++;
            }

            ans = max(ans, len);
        }
    }

    cout << ans << endl;
    return 0;
}
```

#### 10. ACM写法（逐行注释版）

```C++
#include <iostream>          // 输入输出
#include <vector>            // vector 容器
#include <unordered_set>     // 哈希集合 unordered_set
#include <algorithm>         // max 函数
using namespace std;         // 使用标准命名空间

int main() {                 // 程序主函数
    int n;                   // 表示数组中有多少个数字
    cin >> n;                // 读入 n

    vector<int> nums(n);     // 创建长度为 n 的数组 nums
    for (int i = 0; i < n; i++) {   // 循环 n 次读入每个数字
        cin >> nums[i];      // 把输入数字存进 nums[i]
    }

    unordered_set<int> st(nums.begin(), nums.end()); // 把 nums 所有元素放进哈希集合，并自动去重
    int ans = 0;             // 记录最长连续序列长度

    for (int x : st) {       // 遍历集合中的每一个数字 x
        if (!st.count(x - 1)) {  // 如果 x-1 不存在，说明 x 是连续序列起点
            int cur = x;     // cur 表示当前扩展到的数字
            int len = 1;     // 当前这条连续序列的长度，初始至少是 1

            while (st.count(cur + 1)) { // 只要下一个数字还存在，就继续往后扩展
                cur++;       // 当前数字加 1
                len++;       // 长度也加 1
            }

            ans = max(ans, len); // 用当前序列长度更新答案
        }
    }

    cout << ans << endl;     // 输出最终答案
    return 0;                // 程序正常结束
}
```

#### 11. 为什么这题是 `O(n)`

- 每个数插入 set 一次
- 每个数最多被某一条连续序列统计一次
- 查找 `count()` 平均是 `O(1)`

所以总复杂度是：

```TXT
O(n)
```

#### 12. 一句话总结

```TXT
这题的核心不是排序，而是用 unordered_set 快速判断数字在不在；
然后只从连续序列的起点开始往后扩展。
```
