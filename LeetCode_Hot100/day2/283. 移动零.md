# 283. 移动零

## 题意

给定一个数组 `nums`，要求把所有 `0` 移动到数组末尾，同时保持非零元素的相对顺序不变。

限制：

- 必须原地操作
- 不能新开数组

## 核心思路

这题不要想着“怎么搬 0”，而要想着：

**把所有非 0 元素按顺序收集到前面，剩下的位置统一补 0。**

这就自然对应到双指针：

- `right`：负责从左到右扫描整个数组
- `left`：指向“下一个非 0 应该放的位置”

流程：

1. `right` 从左到右遍历
2. 如果 `nums[right] != 0`
   - 把它放到 `nums[left]`
   - `left++`
3. 第一轮结束后，前面已经是按顺序排好的非 0 元素
4. 把 `left` 后面的所有位置都填成 `0`

## 为什么这样做是对的

- `right` 是从左到右扫描的
- 非 0 元素也是按扫描顺序依次写入前面
- 所以非 0 元素的相对顺序不会变
- 又因为没有开新数组，所以满足原地操作

## 例子

输入：`[0,1,0,3,12]`

第一轮把非 0 往前写：

- 看到 `1`，写到前面
- 看到 `3`，接着写
- 看到 `12`，继续写

这时数组前面已经整理成：`[1,3,12,...]`

第二轮把剩余位置补 `0`：

结果：`[1,3,12,0,0]`

## 下标版 C++

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int left = 0;

        for (int right = 0; right < nums.size(); right++) {
            if (nums[right] != 0) {
                nums[left] = nums[right];
                left++;
            }
        }

        for (int i = left; i < nums.size(); i++) {
            nums[i] = 0;
        }
    }
};
```

## 指针版 C++

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int* left = nums.data();
        int* right = nums.data();
        int* end = nums.data() + nums.size();

        while (right < end) {
            if (*right != 0) {
                *left = *right;
                left++;
            }
            right++;
        }

        while (left < end) {
            *left = 0;
            left++;
        }
    }
};
```

## `left`、`*left`、`left++`、`(*left)++` 的区别

这是这题最容易混的地方。

假设：

```cpp
vector<int> nums = {1, 3, 12, 5, 8};
int* left = nums.data() + 3;
```

这时：

- `left` 指向 `nums[3]`
- `*left` 就是 `nums[3]` 这个元素的值，也就是 `5`

### 1. `left`

`left` 是一个指针变量，里面存的是地址。

### 2. `*left`

`*left` 是解引用，表示“取出 `left` 当前指向的那个元素”。

例如：

```cpp
*left = 0;
```

意思是：把 `left` 指向的位置改成 `0`。

执行后数组变成：

```cpp
{1, 3, 12, 0, 8}
```

注意：

- 被改的是数组里的元素
- 不是 `left` 这个指针本身

### 3. `left++`

让指针往后移动一格。

如果原来 `left` 指向 `nums[3]`，执行后就指向 `nums[4]`。

### 4. `(*left)++`

这不是移动指针，而是把当前元素值加 1。

如果 `*left` 原来是 `5`，执行后会变成 `6`。

### 5. 这一组可以直接对照记忆

```cpp
left++      // 指针往后走
*left = 0   // 把当前指向的元素改成 0
(*left)++   // 把当前指向的元素值加 1
```

## 为什么 `nums.data() + nums.size()` 可以表示末尾

这里的 `end` 不是最后一个元素，而是 **尾后位置**。

对于：

```cpp
vector<int> nums = {10, 20, 30, 40};
int* p = nums.data();
```

有：

- `p` 指向 `10`
- `p + 1` 指向 `20`
- `p + 2` 指向 `30`
- `p + 3` 指向 `40`
- `p + 4` 指向最后一个元素后面的位置

而 `nums.size()` 正好是 `4`，所以：

```cpp
int* end = nums.data() + nums.size();
```

就是尾后位置。

可以把它理解成 C++ 常见区间：

```cpp
[begin, end)
```

左边包含，右边不包含。

图示：

```text
nums:   [10] [20] [30] [40]
         ^    ^    ^    ^    ^
         p   p+1  p+2  p+3  p+4
                          last end
```

注意：

- `p + 3` 是最后一个元素
- `p + 4` 是尾后位置
- `end` 可以比较，不能解引用

也就是说：

```cpp
right < end   // 合法
right == end  // 合法
*end          // 不合法
```

## 这题里我容易写错的点

### 1. `if(&right == 0)` 是错的

`&right` 表示“指针变量 `right` 自己的地址”，不是数组里的元素。

如果要判断当前指向的值是不是 `0`，应该写：

```cpp
if (*right == 0)
```

或者更常用：

```cpp
if (*right != 0)
```

### 2. `(*right)++` 不是移动指针

```cpp
(*right)++
```

表示：把当前元素值加 1。

如果要让指针继续往后走，应该写：

```cpp
right++
```

### 3. `*left = &right` 是错的

左边 `*left` 是一个 `int`，右边 `&right` 是一个地址，类型不对。

这里真正想做的是：

```cpp
*left = *right;
```

意思是把 `right` 当前指向的那个数，复制到 `left` 指向的位置。

### 4. `(*left)++` 和 `left++` 完全不是一回事

- `left++`：移动指针
- `(*left)++`：修改当前元素值

## 伪代码

```text
left = 0

for right 从 0 到 n-1:
    如果 nums[right] != 0:
        nums[left] = nums[right]
        left += 1

for i 从 left 到 n-1:
    nums[i] = 0
```

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 这题一句话总结

**不要想着搬 0，而是用双指针按顺序收集非 0 到前面，最后把剩余位置补 0。**
