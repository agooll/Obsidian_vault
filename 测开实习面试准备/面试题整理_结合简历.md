# 面试题整理｜结合简历高频追问版

## 使用说明

- 这份不是全量题库，而是结合你的简历内容，从 `测试开发/面试题` 里筛出来的高命中题
- 优先级判断逻辑：
  - 简历里明确写过的内容，优先级最高
  - 项目里真正做过的内容，优先级高
  - 可能被顺着追问的基础题，优先级中等
- 建议复习顺序：

```text
WMS项目 -> 接口测试/自动化 -> MySQL -> Linux -> 性能测试 -> 计网 -> Python
```

## 一、你最可能被问到的主线

你的简历最容易被面试官围绕这几条主线展开：

- `WMS 仓库管理系统` 怎么做测试
- `JMeter` 怎么做接口测试、接口回归、参数化、关联
- `SQL / MySQL` 怎么做数据校验、造数、查问题
- `Linux` 下怎么看日志、资源、进程
- `性能测试` 怎么压、怎么看指标、怎么定位瓶颈
- `Python` 你懂到什么程度，是否能支撑自动化测试
- `测试流程` 是否真的理解，而不是只会工具操作

## 三、高频：接口测试 / 接口自动化 / JMeter

这部分和你的简历匹配度也很高。

### 1. 你为什么用 JMeter 做接口测试

- 熟悉 HTTP/HTTPS 场景
- 适合接口联调、回归和并发压测
- 支持参数化、断言、提取器、监听器

### 2. 你怎么实现 token 提取和后续传参

- JSON 提取器
- 正则表达式提取器
- Header 中引用变量

### 3. JMeter 里怎么处理接口关联

- token
- order_id
- 入库单号
- 库存 ID

### 4. JMeter 里常用哪些断言

- 响应断言
- JSON 断言
- 状态码断言
- 关键字段值断言

### 5. 一个接口响应结果每次都不一样，你怎么断言

- 断固定字段
- 断业务码
- 断响应结构
- 断关键字段存在性

### 6. JMeter 线程组里的线程数、启动时间、循环次数分别是什么意思

- 线程数：并发用户数
- 启动时间：这些线程多久内全部拉起
- 循环次数：每个线程执行多少轮

### 7. 你怎么做 CSV 参数化

- 为什么要参数化
- 什么字段适合参数化
- 如何避免重复数据污染

### 8. 你怎么生成批量测试数据

- CSV
- SQL
- 存储过程

### 9. JMeter 中 cookie、session、token 有什么区别

- cookie / session 更偏会话保持
- token 更偏前后端分离鉴权

### 10. HTTP 和 HTTPS 有什么区别

- HTTP 明文
- HTTPS 有 TLS/SSL 加密
- 端口不同
- 安全性不同

### 11. 常见 HTTP 状态码有哪些

- 200
- 201
- 400
- 401
- 403
- 404
- 500

### 12. GET 和 POST 的区别

- 参数位置
- 语义不同
- 缓存行为不同
- 安全性不能简单说“POST 一定安全”

### 14. 你怎么理解 fixture

- 用来做前置和后置
- 可以复用登录态、环境准备、数据清理
- 常见 scope：function / class / module / session

- 不能改生产数据

## 四、高频：MySQL / SQL / 数据校验

你简历里明确写了 SQL 校验、存储过程造数、慢查询分析，这里很容易被问。

### 1. 你在项目里数据库主要拿来做什么

- 校验接口结果
- 校验库存变更
- 构造测试数据
- 辅助 bug 定位
- 支撑性能测试

### 2. 你怎么做数据库校验

- 接口调用后查表
- 对比关键字段
- 校验状态、数量、业务结果是否正确

### 3. 你如何验证入库 / 出库 / 盘点的数据一致性

- 入库后库存是否增加
- 出库后库存是否扣减
- 盘点后库存是否修正
- 异常流程下数据库是否被错误修改

### 4. 你什么时候会用存储过程

- 大量批量造数据时
- 用 `CALL` 快速生成用户、商品、库存等数据

### 5. delete / truncate / drop 的区别

- `delete` 删除数据，不重置自增
- `truncate` 清空表，重置自增，不带 where
- `drop` 连表结构一起删

### 6. where 和 having 的区别

- where 是分组前过滤
- having 是分组后过滤

### 7. 内连接、左连接、右连接的区别

- inner join 取交集
- left join 保留左表全部
- right join 保留右表全部

### 8. 你用过哪些常见数据类型

- int
- varchar
- char
- datetime

### 9. 你遇到过什么数据库相关 bug

- 字段长度设计不合理
- 状态值为空或脏数据
- 逻辑删除 / 物理删除设计不合理
- 索引缺失导致查询慢

### 10. 慢查询你怎么分析

- 先看慢查询日志
- 看是不是 SQL 本身有问题
- 看索引是否缺失
- 看数据量和执行计划

### 11. 项目里什么场景要建索引

- 高频查询字段
- 关联字段
- 排序字段
- 组合查询字段

### 12. 你怎么理解事务

- 一组操作要么都成功，要么都失败
- 对测试来说，重点看异常情况下是否回滚

### 6. 如何查看 mysql 进程

### 7. 如何查看某个端口是否被占用

你的简历里明确写了 `100 - 300 并发`、`响应时间 / 吞吐量 / 错误率`、`慢查询分析`，这里很容易被深挖。

### 1. 你做过什么性能测试

- 哪些接口压过
- 并发范围多少
- 为什么选这些场景压

### 2. 你关注哪些核心性能指标

- 响应时间
- 吞吐量
- 错误率
- TPS / QPS

### 3. 你怎么设计压测场景

- 登录
- 入库
- 出库
- 库存查询
- 混合场景还是单接口场景

### 4. 你为什么选 100 - 300 并发

- 是模拟实际业务量
- 还是当前机器和项目阶段的能力验证

这个问题你最好提前准备好自己的解释。

### 5. 压测前你做了哪些准备

- 造数据
- 参数化
- 保证 token / 用户 / 商品可用
- 确认环境稳定

### 6. 性能瓶颈你怎么定位

- 先看 JMeter 结果
- 再看 Linux 资源
- 再看服务日志
- 再看 MySQL 慢查询

### 7. 如果响应时间升高，你怎么判断是哪里的问题

- 应用层
- 数据库层
- 网络层
- 压测脚本自身

### 8. 什么是长稳测试 / 稳定性测试

- 长时间运行
- 观察错误率是否上升
- 看资源是否持续增长
- 查是否有内存泄漏、连接泄漏

### 9. TPS 和响应时间是什么关系

- TPS 高不代表系统一定健康
- 还要结合响应时间和错误率一起看

### 10. 如果压测时数据库 CPU 很高，你会怎么分析

- SQL 慢
- 索引问题
- 数据量问题
- 并发过高导致连接压力
