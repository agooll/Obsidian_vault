# 自我介绍｜测试实习生

## 使用说明

- 主讲版本优先使用下面的 `3分钟版`
- 如果面试官要求简短一点，使用 `1分钟版`
- 如果岗位偏智能软件 / 仿真 / 算法验证，重点看 `岗位定制表达`
- 目标不是背稿，而是记住自己的主线：

```text
我是谁 -> 我做过什么测试 -> 我为什么适合这个岗位 -> 我接下来想往哪里深入
```

## 一、3分钟版

面试官您好，我叫何宇乾，目前是广东科技学院软件工程专业本科在读，想应聘测试开发实习生或软件测试实习生。

我平时主要在往测试开发方向积累，一方面系统学习了软件测试、数据库、操作系统、计算机网络这些基础课程，另一方面也做了比较完整的项目实践。我的技术栈主要集中在 Python、MySQL、Linux 和 JMeter，也会用 SQL 做数据校验，能围绕需求拆测试点、写测试用例、做接口测试、回归测试和基础性能测试。

我最近做得比较完整的一个项目是 WMS 仓库管理系统测试实践。我基于本地搭建的开源 WMS 环境，围绕登录鉴权、入库、出库、库存查询、盘点、权限管理这些核心模块开展测试，梳理了 80 多个测试点，编写了 100 多条功能和接口测试用例，重点覆盖正常流程、异常流程、边界值和权限校验。后面我还用 JMeter 做了接口回归脚本，里面包括 token 提取、接口依赖、CSV 参数化、响应断言和批量数据驱动；做并发压测时，我会结合 Linux 资源、服务日志和 MySQL 慢查询去看瓶颈，并且用 SQL 校验库存数据一致性。这个项目让我对测试流程、接口链路、数据校验和质量闭环有了比较完整的理解。

除了业务测试项目，我还做过一个 SafetyHelmetAI 的实时安全帽检测 Demo。这个项目更偏智能软件和性能验证，我参与了数据整理、模型训练、截图链路性能对比、不同推理后端的延迟和稳定性测试，以及端到端闭环验证。这个经历让我对“功能是否可用”之外的性能、稳定性和验证方法有了更直观的认识，也让我对智能软件、仿真验证这类方向更感兴趣。

另外，我平时也会借助 Codex、Cursor 这类 AI 工具辅助脚本编写、Bug 定位和文档整理，但我理解它们本质上是提高效率的工具，真正关键的还是测试思路、问题定位能力和对业务结果的判断。

我了解到这类岗位通常会比较看重对岗位和业务的理解。如果岗位和智能软件、仿真或自动化验证相关，我会觉得这个方向和我的兴趣比较匹配，因为我本身既做过传统的接口和回归测试，也接触过视觉项目里的性能和稳定性验证。我希望在实习中继续把自动化测试、质量保障和更深入的问题分析能力做扎实，也愿意在有余力的情况下继续学习白盒测试相关内容。我的回答完毕。

## 二、1分钟版

面试官您好，我叫何宇乾，是广东科技学院软件工程专业本科在读，目前主要在往测试开发方向积累。我的基础主要是 Python、MySQL、Linux 和 JMeter，能做测试点拆解、测试用例编写、接口测试、回归测试、基础性能测试和 SQL 数据校验。

我做过一个比较完整的 WMS 仓库管理系统测试项目，围绕登录、入库、出库、库存和权限这些核心模块，设计了 80 多个测试点、100 多条测试用例，并用 JMeter 做了接口回归和并发压测，还会结合 SQL、Linux 日志和慢查询去分析问题。除此之外，我还做过 SafetyHelmetAI 这类偏智能软件的项目，接触过模型验证、推理性能对比和端到端稳定性测试。所以我希望进入测试实习岗位后，继续把自动化测试、质量保障和问题定位能力做得更扎实。

## 三、岗位定制表达

### 1. 如果岗位偏测试开发 / 接口自动化

可以强调：

- 我对接口测试和回归测试这一块积累会更多一些
- 我做过 token 提取、接口依赖、CSV 参数化、响应断言和数据驱动
- 我比较关注测试闭环，不只是接口通不通，也会看数据库结果和业务数据一致性

### 2. 如果岗位偏性能测试 / 稳定性测试

可以强调：

- 我做过基础并发压测，会关注响应时间、吞吐量、错误率这些指标
- 我会结合 Linux 资源、服务日志和 MySQL 慢查询一起看问题
- 我理解性能测试不只是压数据，更重要的是定位瓶颈和验证系统稳定性

### 3. 如果岗位偏 AI / 仿真 / 智能软件验证

可以强调：

- 我除了传统测试项目，还做过 SafetyHelmetAI 这种偏智能软件的验证项目
- 我接触过模型训练、截图链路、推理后端对比和端到端闭环验证
- 我对仿真验证、性能验证、稳定性验证这类方向比较感兴趣
- 我理解这类岗位不只是测功能，还要验证算法行为、性能表现和整体可靠性

可直接补一句：

> 我比较理解为什么这类智能软件需要仿真环境或者高效验证手段，因为很多复杂行为如果只靠人工反复验证，成本会比较高，所以我对这类岗位会更有兴趣。

## 五、表达原则

### 1. 主项目顺序

- 主项目先讲 `WMS`
- 第二项目再讲 `SafetyHelmetAI`
- 不要两个项目平均发力

### 2. 说法要像自己做过

优先说：

- 我做过什么
- 我重点关注什么
- 我怎么验证
- 我从项目里学到了什么

少说：

- 我都会
- 我很精通
- 我全面负责所有内容

### 3. AI 工具不要抢主线

可以说：

- 我会用 AI 工具提升效率

但不要说成：

- 我的核心能力是会用 AI

因为核心还是测试思路、验证方法和问题定位能力。

---

## 六、容易踩坑的地方

- 不要一上来就从学校经历开始流水账
- 不要把自我介绍讲成简历逐条朗读
- 不要过度强调“我什么都懂”
- 不要把 AI 工具当成主卖点
- 不要把 SafetyHelmetAI 讲得比 WMS 还重，除非岗位明显偏智能软件验证
