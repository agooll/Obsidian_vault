# 电子商务网站答辩速背版

## 1. 项目简介
这是一个用 HTML、CSS、JavaScript 原生技术实现的电商网站前端项目，主要模拟商品展示、分类筛选、购物车、登录注册、订单生成、主题切换和在线客服等典型业务功能。

## 2. 技术栈
- HTML：页面结构
- CSS：页面样式、响应式布局、主题切换
- JavaScript：数据渲染、事件交互、状态管理、本地存储

## 3. 核心功能
- 首页商品展示
- 分类页筛选、搜索、排序
- 购物车增删与总价计算
- 登录注册校验
- 结算生成模拟订单
- 秒杀倒计时
- 在线客服
- 多主题切换
- 响应式适配

## 4. 核心实现
- 商品数据放在 `data.js`
- 状态统一放在 `state` 对象
- 页面通过渲染函数动态生成商品列表
- 用 `localStorage` 保存购物车、用户、订单、主题和倒计时
- 用 `sessionStorage` 保存广告关闭状态

## 5. 项目亮点
- 不依赖框架，使用原生 JS 完成交互
- 功能模块较完整
- 有本地持久化能力
- 有响应式布局
- 有主题切换和客服等增强交互

## 6. 项目不足
- 没有后端接口
- 没有真实支付和数据库
- 搜索功能较基础
- 客服是规则回复，不是智能客服

## 7. 高频问题快答
### 为什么做电商项目？
因为电商模块完整，适合综合展示前端页面、交互和状态管理能力。

### 为什么用原生 JS？
为了更直接体现 DOM 操作、事件处理和前端基础能力。

### 购物车怎么实现？
数据保存在 `state.cart` 和 `localStorage` 中，加入商品后重新计算数量和总价并刷新视图。

### 为什么刷新后登录状态还在？
因为用户信息保存到了 `localStorage`，页面初始化时会重新读取。

### 主题切换怎么实现？
切换 `body` 的 `data-theme`，再通过 CSS 变量控制整体配色。

### 如果继续优化会做什么？
接后端、加商品详情页、做真实订单系统、优化搜索和组件化开发。

## 8. 收尾总结
这个项目重点不是后端交易，而是通过原生前端技术完成一套比较完整的电商交互流程，体现我对页面搭建、交互实现和状态管理的掌握。
