# Obsidian 笔记整理 Skill（可复用）

## 适用场景

- 从网页/飞书/cpdown 导出的 Markdown 样式混乱。
- 需要“去头去尾”，保留中间正文内容。
- 需要统一成你的笔记风格：多级标题 + 紧凑排版 + 可读列表。
- 需要把 JMeter / 抓包类知识点写入到对应章节。

## 强约束（必须遵守）

1. 默认只改 Markdown 文件。
2. 不改 `.obsidian`、`.git`、`.trash`。
3. 不删除正文知识点，除非用户明确要求。
4. 大改前先给修改计划。
5. 批量重命名、移动前先列清单。
6. 学习笔记优先按“概念、问题、原因、解决方式、示例、面试表达”整理。
7. 测试开发笔记优先保留实战问题和排错过程。

## 标准处理流程（SOP）

### 第 1 步：识别并去除噪音

仅清理“头尾噪音”，不动中间正文：

- 头部常见噪音：重复标题、外部标识、最近修改、头像链接、AI 速览提示。
- 尾部常见噪音：评论区、推荐内容、站点底部菜单、客服/帮助中心等。
- 删除包裹全文的错误代码围栏（如 ` ```md `）。

### 第 2 步：保留正文并修复显示

- 保持正文知识点不删减。
- 清理零宽字符与异常空白字符。
- 合并多余空行，保留必要段落间隔。
- 将 `• / ◦ / ▪` 统一为 Markdown 列表。
- 将断开编号（如单独一行 `1.`）合并为正常有序列表。

### 第 3 步：标题层级整理（按你的风格）

- 一级主标题：`#`
- 章节标题（如 `1 性能测试介绍`）：`##`
- 小节标题（如 `1.1 概述`）：`###`
- 段内关键小标题（如“概念”“需求来源”“面试题”）：`####`
- 若某标题无内容，取消该标题，避免空壳结构。

### 第 4 步：AI 概览单独成块

若有“内容由 AI 生成”摘要：

- 放在文首独立区域：`## AI 概览`
- 子标题统一：`### 来源说明`、`### 本文主题`、`### 关键要点`
- 摘要内容改为紧凑编号列表。

## 输出规范（每次执行后）

- 必须告知：`文件路径 + 关键修改点所在行号`。
- 必须说明：是否只做了样式优化、是否删改正文内容。
- 如果用户要更紧凑，给两档选项：
  1. 平衡版（推荐）
  2. 超紧凑版（更密集）

## 快速执行口令（给 AI）

```text
按“Obsidian笔记整理Skill”处理 <文件路径>：
1) 只去头去尾噪音，中间正文不删；
2) 修复 Markdown 显示（列表、空行、编号）；
3) 按现有内容补多级标题，空标题不要保留；
4) 如有“AI生成内容”，单独做 AI 概览块。
```

## 本次会话沉淀的关键偏好

- 优先“保留原文内容”，只做结构与显示优化。
- 风格要与既有笔记一致，不能过度重写。
- 标题必须“有内容”，否则不单独成标题。
- 每次修改后要明确定位到第几行。
